Python 描述器深入简出

|阅读量:

字典查找优先级

在Python中有两类描述器，一种叫“数据型描述器”，另一种自然便是“非数据型描述器”。非数据型描述器则只定义了__get__，常见的非数据型描述器包括所有的方法（普通，静态，类方法）；而数据型描述器至少定义了__set__或__delete__中的一个方法。在做出形如a.desc的时候获取的值由下面的顺序执行：

  1. 调用定义了__get__方法的数据型描述器的__get__并拿到其返回值
  2. 在实例字典上找对应的值
  3. 获取非数据类型描述器的__get__返回值
  4. 不是描述器，在mro上按顺序查找__dict__
  5. 报错AttributeError

方法

所有的函数同时也是描述器，其描述器的一面体现在使用实例获取类体内定义的函数时会获取到方法（自动添加self作为第一个参数）。为了简便性我将创建一个包装现有函数的Function描述器，和Method类表示方法。

主要的实现思路就是当当实例尝试获取一个函数的时候，返回一个Method，这和Overload里的BoundOverloadDispatcher有异曲同工之妙——描述器只能管获取属性，而对属性进行任何操作描述器都管不到，所以便需要一个中间人知道所有的信息并处理__call__方法。
而方法类就由此而生了，它的主要职责就是在__call__的时候把实例放到参数列表的第一位，使得感觉上方法和函数有本质上的区别，实则不然。这也就解释了为什么可以用A.bar("desc", "hi")的方式也可以成功调用——A.bar返回的是真的函数自身，而函数本身需要两个参数，只是需要手动输入第一个参数而已。就这个案例而言，第一个参数甚至不需要是A类的实例。
